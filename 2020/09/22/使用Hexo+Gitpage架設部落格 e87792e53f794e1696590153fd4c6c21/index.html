使用Hexo+Gitpage架設部落格

發表於 2020-09-22 更新於 2020-09-24 Disqus：

OS環境

macOS 10.15.6

需要的環境

Homebrew
git
node.js
hexo

安裝步驟

安裝Homebrew(此為mac的套件管理工具，其他OS可能用會用別的)

安裝git

安裝node.js

安裝hexo

建立Hexo部落格

建立一個資料夾，或是直接 cd 到已經建立好的中 hexo init

(記得把rootFolder改成自己想要的資料夾名稱)

此時已經建立好Blog的架構了

node_modules：相依套件存放位置

package.json：相依套件列表

public：生成的靜態網站資源

scaffolds：鷹架資料夾。建立新文章時，會根據 scaffold 來建立檔案。

source：部落格原始檔案放置的位置，文章會放在source/_posts底下。

themes：主題資料夾，上圖的landscape跟next就是兩個主題。Hexo 會根據主題來產生靜態檔案。社群上有很多樣式供選擇。

_config.yml：網站配置檔案，大部分的設定都在該檔案中。

常用指令

接著執行建置跟啟動，就可以在 http://localhost:4000/ 看到Blog 。

之後會透過部署的指令 hexo deploy (hexo d) 將網站部署到 github上。

預設的樣式是landscape

設定網站

修改網站設定檔 _config.yml

詳細的設定可以參考這裏

比如說要修改語系 language: zh-tw

修改主題樣式

Hexo有眾多Theme可以選擇，切換Theme只需要幾個步驟。

  1. 在網路上找到想要的Hexo Theme
  2. 把檔案放到/themes/目錄下
  3. 修改 _config.yml 中的 theme: <資料夾名稱>

舉例來說，要將主題換成Next(許多人使用的主題之一)

  1. 下載主題(此處使用git clone)

  1. 修改根目錄的網站設定 _config.yml ，將內容改為theme: next 。

  2. 重新啟動server

此時就可以看到已經換了一個樣式。

將部落格部署到Gitpage上

  1. 在GitHub 創建一個名稱為 <username>.github.io 的專案(<username>必須跟github帳號一樣)
  2. 要把Hexo deploy到Github上面，需要安裝
  1. 在網站設定 _config.yml ，修改 repository 位置
  1. 執行 hexo d 即可發佈到 GitHub 上

此時連到github上會看到檔案已經更新，可以連上 <username>.github.io看成果。

後記

由於我原本的需求是要從外部直接產出md格式的內容，直接放到部落格中，而不是使用hexo來建立文章，所以把新增文章相關的指令寫在最下方。
